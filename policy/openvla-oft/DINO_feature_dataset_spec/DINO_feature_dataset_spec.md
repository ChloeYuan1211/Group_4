# DINO 特征数据集构造说明

## 1. 目标

当前阶段只构造数据集，不讨论损失函数。

每个样本定义为：

\[
(x,\; F^{GT})
\]

其中：

- `x`：原图，不做平移，不做翻转
- `F^{GT}`：由原图的多个几何变换版本经过 DINO 提取、反向对齐、再做有效区域平均后得到的目标特征

推荐保存形状为：

\[
F^{GT} \in \mathbb{R}^{16 \times 16 \times D_{dino}}
\]

如有需要，也可以再展平为：

\[
F^{GT} \in \mathbb{R}^{256 \times D_{dino}}
\]

---

## 2. DINO 输出表示

对一张输入图像 `x`，经过 DINO 后，取 patch token 的输出特征：

\[
F^{(0)} = f(x) \in \mathbb{R}^{16 \times 16 \times D_{dino}}
\]

说明：

- `16 × 16` 表示 patch 网格
- `D_dino` 表示每个 patch token 的特征维度
- 这里只讨论 patch-level feature
- 不包含 `CLS token`
- 不包含 `register token`

---

## 3. 几何变换集合

对同一张原图 `x`，构造 5 个几何变换版本：

\[
\mathcal{T} = \{T_{\text{up}},\; T_{\text{down}},\; T_{\text{left}},\; T_{\text{right}},\; T_{\text{flip}}\}
\]

具体包括：

- 上平移
- 下平移
- 左平移
- 右平移
- 左右翻转

对于每个变换后的图像 `T_m(x)`，经过 DINO 得到：

\[
F^{(m)} = f(T_m(x)) \in \mathbb{R}^{16 \times 16 \times D_{dino}}
\]

---

## 4. 图像平移规则

平移时采用如下规则：

- 如果原图向右平移一小段，则原图右边一小段裁掉
- 左边新出现的区域填充纯白色
- 其他方向平移同理

即：

- 右移：右裁剪，左补白
- 左移：左裁剪，右补白
- 上移：上裁剪，下补白
- 下移：下裁剪，上补白

---

## 5. 平移尺度约束

由于最终 DINO 输出是 `16 × 16` 的 patch 网格，因此平移必须与 patch 网格严格对齐。

在当前设置下：

- 输入图像大小：`224 × 224`
- patch 网格大小：`16 × 16`

所以每个 patch 对应：

\[
224 / 16 = 14 \text{ pixels}
\]

因此平移量必须取 patch 尺度的整数倍：

\[
k \times 14 \text{ pixels}
\]

也就是：

- 平移 1 格 = 14 像素
- 平移 2 格 = 28 像素
- 依此类推

这样才能保证输出的 `16 × 16 × D_dino` 特征可以严格按格子对齐。

---

## 6. 特征反向对齐规则

对每个变换分支得到的特征 `F^(m)`，不直接参与平均，而是先对齐回原图的 patch 坐标系，得到：

\[
\tilde{F}^{(m)} \in \mathbb{R}^{16 \times 16 \times D_{dino}}
\]

对齐规则如下：

| 图像上的变换 | 对齐回原图时的操作 | 对齐后无效区域 |
|---|---|---|
| 向右平移 `k` 格 | feature 整体向左移 `k` 格 | 右侧 `k` 列无效 |
| 向左平移 `k` 格 | feature 整体向右移 `k` 格 | 左侧 `k` 列无效 |
| 向下平移 `k` 格 | feature 整体向上移 `k` 格 | 下侧 `k` 行无效 |
| 向上平移 `k` 格 | feature 整体向下移 `k` 格 | 上侧 `k` 行无效 |
| 左右翻转 | feature 再做一次左右翻转 | 无无效区域 |

例子：

- 原图向右平移 1 个 patch
- 得到右移后的 feature map
- 对齐回原图时，将该 feature map 向左平移 1 格
- 此时右边会空出 1 列
- 这 1 列不参与后续平均

---

## 7. 有效区域 Mask

为了正确处理“空白列 / 空白行不参与平均”，每个分支除了对齐后的 feature，还要维护一个有效区域 mask：

\[
M^{(m)} \in \{0,1\}^{16 \times 16}
\]

定义：

- `M^(m)[i,j] = 1`：该 patch 位置有效
- `M^(m)[i,j] = 0`：该 patch 位置是对齐后空出来的无效区域

说明：

- 无效区域在存储 tensor 时可以填 0
- 但这个 0 只是占位值
- 它不代表真实 feature
- 它不能像正常 feature 一样直接参与平均

---

## 8. 平均规则

最终目标特征 `F^{GT}` 不是普通平均，而是按有效区域做 masked average。

对每个 patch 位置 `(i,j)`：

\[
F^{GT}[i,j] =
\frac{
\sum_m M^{(m)}[i,j] \cdot \tilde{F}^{(m)}[i,j]
}{
\sum_m M^{(m)}[i,j]
}
\]

含义：

- 只对有效分支求和
- 分母只统计有效分支数
- 无效分支既不进分子，也不进分母

因此：

- 中心区域通常由 5 个分支共同平均
- 边界区域参与平均的分支数会减少
- 角落区域参与平均的分支数会进一步减少
- 左右翻转分支在所有位置都有效

---

## 9. 平均时必须注意的细节

这是当前方案最关键的部分。

### 9.1 空白列和空白行不能当作真实 0 参与平均

例如：

- 原图右移 1 格
- 对齐回来后右边空出 1 列

这一列可以在 tensor 中填 0，但：

- 这个 0 不能和其他有效 feature 一起做普通平均
- 否则会把均值错误拉低

正确做法是：

- 用 mask 指示无效区域
- 平均时只统计有效位置

### 9.2 不同 patch 位置参与平均的分支数不同

例如平移 1 格时：

- 中间区域可能有 5 个有效分支
- 上边缘会缺少“上移分支”
- 下边缘会缺少“下移分支”
- 左边缘会缺少“左移分支”
- 右边缘会缺少“右移分支”
- 四个角落通常有效分支更少

因此你的最终目标特征并不是全图统一由 5 个分支平均出来的，而是：

**每个 patch 位置只由对它有效的那些分支平均得到。**

---

## 10. 左右翻转分支的处理

对于左右翻转分支：

1. 对原图做左右翻转
2. 经过 DINO 得到对应 feature map
3. 再对 feature map 做一次左右翻转
4. 即可还原到原图 patch 坐标系

该分支特点：

- 没有空白区域
- mask 全为 1
- 在所有位置都参与平均

---

## 11. 最终数据项

当前阶段只造数据集，不考虑损失，推荐每条数据至少保存以下内容：

```python
{
    "image": x,
    "feature_gt": F_GT
}
```

其中：

- `image`：原图
- `feature_gt`：对齐并平均后的目标特征，建议 shape 为 `[16, 16, D_dino]`

---

## 12. 推荐附加保存信息

虽然当前不讨论损失，但为了后续更稳妥，建议额外保存：

```python
{
    "image": x,
    "feature_gt": F_GT,
    "valid_count": C
}
```

其中：

\[
C[i,j] = \sum_m M^{(m)}[i,j]
\]

表示该 patch 位置最终由多少个有效分支平均得到。

这样后续无论设计什么监督形式，都能知道每个位置的 target 可靠程度。

---

## 13. 数据集构造流程

对每张原图 `x`，完整流程如下：

1. 保留原图 `x`
2. 生成 5 个几何变换版本：上、下、左、右平移，以及左右翻转
3. 每个变换图输入 DINO，得到 `16 × 16 × D_dino` 的 patch feature
4. 将每个分支的 feature 按已知几何关系反向对齐到原图 patch 坐标系
5. 为每个分支构造对应的有效区域 mask
6. 对所有对齐后的 feature 做 masked average
7. 得到最终目标特征 `F^{GT}`
8. 保存为一个样本 `(x, F^{GT})`

---

## 14. 一句话总结

当前要构造的数据集可以定义为：

**原图 `x` 与其对应的 patch-level DINO 目标特征 `F^{GT}` 的配对数据集。**

其中 `F^{GT}` 不是原图直接经过 DINO 得到的单次输出，而是：

**原图经过上、下、左、右平移以及左右翻转后的多个 DINO patch feature，在原图 patch 网格上反向对齐，并通过有效区域 mask 做平均后得到的 `16 × 16 × D_dino` 稳定特征表示。**
